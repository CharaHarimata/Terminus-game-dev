~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
[FUNCTION]
initSetup(<port>,<html>,<script>)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

This will establish your file system, and trigger the server to start communicating with clients.

~~~~~~~~~~~~~~
[REQUIREMENTS]
~~~~~~~~~~~~~~

~port = exactly what it sounds like.

~html = your reference file. It'll create a static folder for you, and plop the script in.

~script = gets plopped in the static folder. It basically handles all the client-side transmission.

`````````````````````````````````````````````````````

[FUNCTION]

initPlayer()
~~~~~~~~~~~~

Triggers creation of a websocket for the client to communicate with. All dependencies are packed in to the client, so it's fairly standalone.

~~~~~~~~~~~~~~
[REQUIREMENTS]
~~~~~~~~~~~~~~

None. ^w^
